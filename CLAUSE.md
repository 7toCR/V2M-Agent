## 一、鼓励的使用方式 / Encouraged Uses

我们欢迎并鼓励以下行为：
We welcome and encourage the following:

1. 免费使用本项目进行学习、研究、教学、实验和非商业演示。
   Use the project freely for study, research, teaching, experiments, and non-commercial demos.
2. 在保留来源信息的前提下，Fork、改进并提交 Issue 或 Pull Request。
   Fork, improve, and contribute Issues or Pull Requests while preserving source attribution.
3. 在论文、博客、技术分享、课程讲义或演示材料中引用本项目，并链接回原仓库。
   Reference the project in papers, blog posts, talks, course notes, or demos with a link back to the original repository.
4. 基于本项目构建自己的实验流程、评测脚本或本地部署方案，并明确说明哪些部分来自 NingBGM。
   Build your own workflows, evaluation scripts, or local deployment setups on top of the project and clearly mark what comes from NingBGM.

## 二、禁止的行为 / Prohibited Conduct

以下行为与本项目社区规范不符：
The following behaviors conflict with this project's community rules:

1. 直接复制本仓库的 README、提示词、Agent 配置、流程设计、条款文本或其他原创内容，换名后重新发布，并声称为自己的原创项目。
   Republishing the repository's README, prompts, agent configs, workflow design, clause text, or other original material under a new name while claiming original authorship.
2. 将本项目整体或大部分内容镜像到其他代码托管平台、知识星球、课程平台或文档站点，而不注明原始来源与仓库链接。
   Mirroring the whole repository or substantial parts of it to other code hosting, course, paywalled, or documentation platforms without clear source attribution and a repository link.
3. 将本项目包装为收费课程、商业咨询、代部署服务、SaaS 功能包或售卖资料，而不明确说明其基础来自 NingBGM。
   Packaging the project as a paid course, consulting offer, deployment service, SaaS feature set, or resold material without clearly disclosing that it is based on NingBGM.
4. 冒充项目作者、维护者或官方合作方，或暗示自己拥有本项目的原创署名权。
   Impersonating the author, maintainer, or an official collaborator, or implying original ownership of the project.
5. 在使用本项目调用第三方模型、API 或远程生成服务时，违反相应服务商条款，并将相关风险归责于本项目作者。
   Violating third-party model, API, or remote generation service terms while using this project and attributing that risk to the project author.

## 三、引用与署名规范 / Attribution Guidelines

如果你在以下场景中使用 NingBGM，请按下列方式注明来源：
If you use NingBGM in the following scenarios, attribute it as follows:

| 使用场景 | 建议署名方式 |
| --- | --- |
| 学术论文 / Academic paper | 在方法、致谢或参考文献中注明 `NingBGM` 仓库名称，并附原仓库链接 |
| 技术博客 / Article | 在正文或文末标注“Based on NingBGM”并附原仓库链接 |
| GitHub 仓库 / Public repo | 在 README 中说明继承或参考自 `NingBGM`，并链接原仓库 |
| 产品演示 / Demo | 在演示页、文档或发布说明中标注使用了 `NingBGM` 的公开流程或组件 |
| 商业服务 / Commercial service | 明确区分自研部分与基于 `NingBGM` 的部分，避免误导性宣传 |

## 四、维权与处理方式 / Enforcement

如果你发现疑似违规使用，可以通过以下方式协助维护社区：
If you discover suspected misuse, the community can help by:

1. 在相关仓库或页面指出来源缺失问题并要求补充署名。
   Asking the relevant repository or page to restore missing attribution.
2. 通过平台的滥用举报渠道提交说明。
   Using platform abuse-report channels where appropriate.
3. 联系项目维护者 `7toCR` 说明情况。
   Contacting the maintainer `7toCR` with the relevant details.

对于持续性、恶意或明显冒充原创的行为，项目维护者保留进一步投诉、要求下架或采取其他必要维权措施的权利。
For repeated, malicious, or clearly deceptive reposting or false authorship claims, the maintainer reserves the right to pursue takedown requests or other appropriate responses.
